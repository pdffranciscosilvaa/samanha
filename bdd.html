Banco de Dados

Um banco de dados é uma coleção organizada de informações estruturadas, geralmente armazenadas eletronicamente em um sistema de computador. Ele permite o armazenamento e a organização de dados para facilitar o gerenciamento e o acesso a essas informações.



Banco de Dados - Relacional


Um banco de dados relacional é um tipo de sistema que organiza e armazena dados em tabelas com uma estrutura pré-definida, permitindo o acesso a pontos de dados relacionados entre si. Esses bancos utilizam um modelo formal que define as relações entre os dados, o que facilita consultas rápidas e eficientes.

Banco de Dados - Não Relacional


banco de dados que não utiliza a estrutura tradicional de tabelas compostas por linhas e colunas, característica dos sistemas de banco de dados relacionais. Em vez disso, esses bancos de dados podem armazenar dados em formatos como documentos, chave-valor, grafos ou colunas. O termo "NoSQL" é frequentemente associado a essas tecnologias, que são projetadas para lidar com grandes volumes de dados não estruturados ou semi-estruturados.

O que é um Diagrama DER?


Um diagrama DER (Diagrama de Entidade e Relacionamento) é uma representação gráfica que ilustra as entidades de um sistema e as relações entre elas. Ele é fundamental para modelar bancos de dados, ajudando a identificar como as informações estão interconectadas e como os dados fluem dentro do sistema.


Diagrama DER



O que é um Diagrama MER?


O diagrama MER (Modelo Entidade-Relacionamento) é uma representação gráfica que ilustra as entidades de um sistema e os relacionamentos entre elas. Ele é amplamente utilizado na engenharia de software e no design de bancos de dados para descrever a estrutura de dados de forma conceitual, facilitando a compreensão das informações que precisam ser armazenadas.

O que é dicionário de Dados?


Um dicionário de dados é uma ferramenta ou um documento que define os elementos de dados em um sistema de informação, como bancos de dados ou aplicações. Ele geralmente contém informações detalhadas sobre cada elemento de dados, incluindo:


  • Nome do Dado: Identificação do elemento.

  • Tipo de Dado: Natureza do dado (por exemplo, texto, número, data).

  • Tamanho: A capacidade máxima do dado, se aplicável.

  • Descrição: Explicação do que o dado representa e seu propósito.

  • Restrições: Regras que limitam os valores que o dado pode assumir (por exemplo, se é obrigatório, exclusivo, etc.).

  • Formato: Como o dado deve ser estruturado ou apresentado.

  • Relacionamentos: Como o dado se relaciona com outros dados no sistema.
